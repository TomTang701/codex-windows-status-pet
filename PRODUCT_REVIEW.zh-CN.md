# Codex Windows 状态宠物——产品评审

## 当前定位

这是一个面向重度 Codex 用户、低干扰的外部状态伴侣。它提供活动对话状态、额度窗口、重置次数、设置和托盘恢复路径，不修改 Codex 核心文件，也不向内置宠物注入文字。

## 当前产品契约

- 悬浮窗只显示活动对话数量，不显示计划步骤 `N/M` 文本。
- 设置支持透明度、颜色、字体大小、置顶、锁定、虚拟桌面 X/Y 坐标、窗口尺寸、等比例缩放和 1–10 秒刷新间隔。
- 悬浮窗隐藏或移出屏幕时，托盘是恢复路径。
- API 边界和无界面回归测试定义在 `API_SPEC.md`。
- 菜单定位和额度日期格式化是可独立测试的 API 边界。

## 优势

- 本地数据边界，不使用第三方后端。
- 额度数据源边界仅限本地，并拒绝传播凭据字段。
- 外部架构降低对 Codex 核心行为的影响风险。
- 明确的配置持久化和多显示器坐标支持。
- API 拆分使配置、活动状态、菜单几何和额度格式化无需启动 Tk 即可测试。

## 当前风险

- Windows Tk、托盘、多显示器和 DPI 行为仍需要实体环境测试。
- 托盘在被注入故障后需要继续验证自动重建，才能视为可靠恢复路径。
- 固定尺寸悬浮窗可能裁剪长诊断信息或本地化文字。
- 干净虚拟环境已经通过依赖导入和回归测试；实体混合 DPI 覆盖仍未完成。
- quota-float 的视觉模型值得借鉴，但没有复制其基于令牌的外部额度数据边界。

## 建议的产品优先级

1. 让托盘和屏外恢复路径可观测且能够自愈。
2. 完成 DPI/混合缩放和多显示器测试覆盖。
3. 增加诊断视图或清晰的“打开日志”操作。
4. 保持默认悬浮窗紧凑、聚焦状态；除非明确做成可配置功能，不要把计划步骤详情重新放回主信息行。
