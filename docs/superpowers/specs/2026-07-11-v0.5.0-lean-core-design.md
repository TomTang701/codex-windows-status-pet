# v0.5.0 Lean-Core Simplification Design

## Baseline

- Runtime production Python files: 39
- Runtime production Python LOC: 2,742
- All scripts Python files / LOC: 53 / 3,621
- Test files: 39
- Test LOC: 2,349
- API modules: 32
- Active normative files / LOC: 10 / 957
- Runtime dependencies: 2
- Released baseline: v0.4.2 at `d946ce0a632a559e69604adb10be519ca28bfa38`

## Problem evidence

Consumer inspection confirms:

- `models_api.py` defines dataclasses unused by runtime or tests.
- `window_size_api.py` implements a superseded free/proportional operation and is imported only by `test_window_size_api.py`.
- `resize_session_api.py` implements superseded plus/minus state and is imported only by `test_resize_session_api.py`.
- `quota_provider_api.py` contains only `return parse_quota_payload(payload)`, has one runtime consumer, and adds no provider, IO, auth, or state boundary.
- The current config downgrade fields are produced by `window_scale_api`/`config_api`, not either historical resize module.

## Alternatives

### Keep all compatibility files

Lowest immediate change risk, but preserves dead concepts and tests only their implementation. Rejected because consumer and rollback evidence shows they are not part of the current compatibility path.

### Broadly merge controllers and APIs

Could reduce more files, but application, settings, presentation, lifecycle, refresh, Tk/Windows, transport, config, and parser boundaries still own real responsibilities. Rejected for this phase because it raises regression risk without evidence.

### Focused deletion and one direct parser connection

Recommended. Delete the three zero-consumer modules and their implementation-detail tests. Replace the quota wrapper import/call with `parse_quota_payload`, while moving the wrapper's privacy/malformed fixtures into the parser test suite. Add a negative source-boundary test so these obsolete modules cannot silently return.

## Data and failure paths

Quota remains:

```text
local Codex app-server payload
→ main-window refresh worker
→ parse_quota_payload
→ QuotaState
→ StatusPresentationController
→ StatusRows
```

Malformed/token-bearing data remains:

```text
payload
→ approved-field parser
→ unknown/token fields dropped
→ unavailable or bounded approved snapshot
→ no credential persistence/display
```

Window scaling remains:

```text
window_scale_percent
→ window_scale_api
→ logical compatibility fields + DPI-aware runtime metrics
→ settings/main window
```

The deleted resize modules are not on either path.

## Regression surface

- quota aliases, malformed responses, Reset Credit containers, token/unknown-field exclusion;
- quota state/loading/last-good/stale/unavailable paths;
- 80–200% scale derivation, legacy geometry migration, schema-1 persistence;
- settings Apply/Save/Close/Defaults, Compact, Hide/Show, menu, lifecycle;
- Quality, inventory, package, strict readiness, RC, and CI workflow.

## TDD contract

Before deletion, add `tests/test_lean_core_boundaries.py` asserting:

- the four obsolete module paths do not exist;
- `main_window.py` imports/calls `parse_quota_payload` and does not mention `normalize_snapshot`;
- active API specs do not advertise deleted boundaries.

It must fail against the current tree because all four modules exist. Add the two provider privacy/malformed fixtures to `tests/test_quota_parse_api.py` before deleting the wrapper test; those behavior tests should remain green throughout because the parser already owns the behavior.

## Documentation and metrics

Remove deleted APIs from bilingual API specs and repository structure where present. Record exact before/after files, LOC, API modules, active normative docs, dependencies, and Quality duration in the release evidence. Do not claim a larger reduction than measured.
