# v0.7.0 bilingual UI and manual Compact design

## Approved outcome

v0.7.0 ships English and Simplified Chinese runtime UI, with `en` as the
persisted default, plus a persisted manual `compact` state.  Automatic idle,
hover, activity, and render-driven compaction are removed.  Phase A keeps
documentation bilingual only for pairs registered in `docs/document_manifest.json`.

## Ownership

- `scripts/api/config_api.py` normalizes the additive `language` and `compact`
  settings without migrating legacy `compact_when_idle` to `compact`.
- `scripts/api/localization_api.py` owns the two static translation maps,
  language normalization, and visible missing-key failures.
- `scripts/api/status_snapshot_api.py` receives a language code for visible
  presentation only; its stable row identifiers and battery semantics remain
  unchanged.
- `scripts/ui/settings_dialog.py`, `scripts/ui/context_menu.py`, and
  `scripts/ui/tray_adapter.py` render translated labels while retaining stable
  action identifiers.
- `scripts/ui/main_window.py` owns application of settings, compact geometry,
  and persistence after an explicit context-menu compact toggle.

## Compact decision

`settings["compact"]` is the only normal compact authority.  Startup applies it
after the root and battery are available.  Explicit menu toggles update the
setting, visual state, and atomic persistence immediately.  Status rendering,
hover, activity, menus, Hide/Show, tray Show, and Settings language preview do
not select a compact state.

## Documentation gates

`scripts/check_doc_privacy.py` scans tracked Markdown only.  It rejects local
absolute paths, file URIs, UNC paths, escaping relative links, and supported
documentation that recommends the retired external launcher.  It permits
repository-relative paths and generic environment-variable paths.

`scripts/check_doc_navigation.py` reads the manifest and verifies reciprocal
language switches and same-language managed-document links.  It does not
require Goal, plan, archive, evidence, or unregistered documents to have a
translation.

## Design verification

The design keeps quota identity, battery source/no-fallback semantics, five row
identities, row visibility, settings transactions, display recovery, Shell
identity, tray ownership, and lifecycle boundaries intact while changing only
the approved language and compact behavior.  The approved product decisions
are implementable through existing narrow owners without a renderer, settings,
or localization framework.  **PASSED.**
