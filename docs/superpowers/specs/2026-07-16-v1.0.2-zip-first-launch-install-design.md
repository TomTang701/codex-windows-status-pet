# v1.0.2 ZIP First-Launch Installation Design

**Date:** 2026-07-16  
**Target release:** v1.0.2  
**Repository:** `TomTang701/codex-windows-status-pet`

## Goal

Support both normal installation paths without duplicating the application download:

1. Run the public GitHub Release bootstrap to install or upgrade the latest stable release.
2. Download the versioned product ZIP, extract it, and run `launch.cmd` to install and launch that exact extracted version.

The ZIP path must no longer fail with a briefly visible console when `runtime.json` has not been created yet.

## User Experience

`launch.cmd` remains the visible fallback entry point in the product ZIP.

- When `runtime.json` exists beside the launcher, it delegates to `launch.ps1` and starts the application as it does today.
- When `runtime.json` is absent, it invokes the local installer using the extracted directory as the source.
- A successful first launch installs to `%LOCALAPPDATA%\Programs\CodexStatusPet`, creates the existing Desktop and Start Menu shortcuts, and starts the installed copy.
- A failed first launch keeps the console open when launched interactively and prints a specific repair message and nonzero exit code.
- The local ZIP workflow installs the extracted version. It does not resolve or download a newer GitHub Release.

## Installer Architecture

Extend `install.ps1` with two mutually exclusive input modes:

- Existing verified artifact mode: `-ArtifactPath`, `-Sha256`, and `-ExpectedVersion`.
- New extracted-source mode: `-SourceRoot`.

Artifact mode keeps its current SHA-256 verification and archive expansion behavior. Source mode resolves the supplied directory, reads its `release-manifest.json`, derives the exact expected version from that manifest, and copies the source tree into a private staging directory before installation. Both modes then use the same manifest validation, compatible-Python discovery, dependency installation, backup/rollback, shortcut creation, and application launch path.

Source mode must reject:

- a missing or non-directory source root;
- an invalid or unsupported manifest;
- an entry point or launcher missing from the source;
- prohibited executable runtime material already rejected from source packages;
- a source root equal to or nested inside the installed destination;
- parameters from both installer modes in one invocation.

The installer must exclude installation-generated material such as `runtime.json` and `runtime-packages` when staging an extracted source. This keeps repair behavior deterministic if the user invokes the fallback again from a previously prepared directory.

## Launcher Behavior

`launch.cmd` performs only routing and user-visible error handling:

1. Resolve its own directory with `%~dp0` so paths containing spaces work.
2. If `runtime.json` exists, run `launch.ps1` unchanged.
3. Otherwise run the adjacent `install.ps1 -SourceRoot "%~dp0"`.
4. Propagate the child exit code.
5. On failure, print a concise message and pause only when the launcher owns an interactive console. Automated and nested invocations must be able to opt out of the pause so tests and scripts do not hang.

Installation logic must not be duplicated in the CMD file.

## Documentation and Release Contract

Update English and Simplified Chinese README and installation documentation to present two supported methods:

- Public bootstrap: recommended for latest-version install, verified repair, and upgrade.
- Versioned product ZIP: download `CodexStatusPet-vX.Y.Z-win11-x64.zip`, extract the entire `CodexStatusPet` directory, and run `launch.cmd` to install that exact version.

Documentation must continue to distinguish the versioned product ZIP from GitHub's generated source archives. It must also state that the bootstrap verifies the release checksum automatically, while manual ZIP users should download only from the official GitHub Release and may verify the published `.sha256` sidecar.

Update all authoritative version sources, changelogs, release metadata, and checks from v1.0.1 to v1.0.2. Release assets remain:

- `CodexStatusPet-bootstrap.ps1`
- `CodexStatusPet-v1.0.2-win11-x64.zip`
- `CodexStatusPet-v1.0.2-win11-x64.zip.sha256`
- `install.ps1`

## Scope

This patch changes only installation, launch routing, related tests, documentation, and release/version metadata. It does not change application UI, quota behavior, settings schema, Codex transport behavior, startup registration policy, or dependency versions unless verification proves a compatibility correction is required.
