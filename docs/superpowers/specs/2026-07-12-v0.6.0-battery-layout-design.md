# v0.6.0 5H Battery Indicator and Layout Tightening Design

## Status

**DESIGN VERIFICATION: PASSED.** This design is authorized by the active continuous-delivery Program Goal after the v0.5.5 hard version gate.

## Observed architecture and semantics

The official local app-server payload enters `parse_quota_payload()`, retains `rateLimits.primary.usedPercent` and `resetsAt`, then reaches `build_status_snapshot()` through `StatusPresentationController.render()` and `Pet.render_status()`. `StatusRows` owns exactly five persistent Tk labels using `ROW_IDS`:

```text
activity, progress, primary_5h, weekly, reset_credit
```

`_percent_left()` currently renders `100 - usedPercent`; therefore current `primary_5h` text means **remaining 5H quota**. The battery fill and percentage text will both mean remaining quota. Missing/malformed `usedPercent` remains `--` and renders no invented fill. `stale`, unavailable, and last-good behavior remain owned by the existing snapshot and quota-state paths.

## Approaches considered

| Approach | Result | Decision |
| --- | --- | --- |
| A. Inline ASCII proportion bar in `primary_5h` | Fits the existing string/Label boundary; fixed ASCII width is predictable across Windows DPI; no new widget tree. | Selected. |
| B. Composed Canvas or widget battery | Precise graphical fill but changes the five-label renderer, DPI geometry, and lifecycle surface. | Rejected as disproportionate. |
| C. Emoji/Unicode battery icon | Compact but low granularity and variable Segoe glyph width/color behavior. | Rejected. |

## Selected presentation

Add a pure formatter owned by the status-snapshot boundary. Valid official input produces:

```text
5h [######----] 60% / 18:40
```

The bar has ten ASCII slots. Filled slots and percentage are both the bounded remaining percentage. Filled-slot count uses nearest integer rounding of `remaining / 10`, clamped to 0–10. A valid 0% renders an empty bar; valid 100% renders a full bar. Invalid/unavailable percentage renders the current truthful `5h -- / time` text without a bar.

No sixth row, background worker, animation, provider, storage field, configuration field, or dependency is introduced.

## Layout tightening

Measured current 100% runtime geometry is root `330×138`, requested root height `135`, status frame `256×118`, requested status height `115`, five labels of height `23`, with `horizontal_padding=12`, `vertical_padding=10`, and `face_text_gap=5`. Window height has only 3px requested-size headroom, so shrinking the canonical window dimensions is unsafe and could reintroduce v0.4.1 clipping.

The minimum coherent metric change keeps the canonical window width/height and proportional scale contract unchanged while reducing whitespace around the five-row content:

```text
BASE_HORIZONTAL_PADDING: 12 → 10
BASE_VERTICAL_PADDING:   10 → 8
BASE_FACE_TEXT_GAP:       5 → 3
```

The content-fit authority will verify every supported scale (80–200 by 5) at DPI 96 and 120 with the longest v0.6.0 `primary_5h` string and existing reset-credit maximum line. Existing special safe vertical values will be retained or adjusted only if the measured all-scale RED requires it.

## Regression and error contract

- Maintain exact five label identities and objects.
- Preserve current color, stale, unavailable, malformed, and last-good quota presentation.
- Preserve mixed-DPI startup position persistence, valid/off-screen recovery, Shell identity, Hide/Show, Compact/Expand, settings lifecycle, drag/restart, lock, and topmost behavior.
- Run pure semantics RED/GREEN, all-scale content-fit RED/GREEN, DPI 96/120 and broader runtime geometry tests, physical-topology admission/mixed-DPI tests, Shell identity tests, Quality, package smoke, and formal RC.

## Design Verification answer

**Yes.** The design improves 5H presentation and tightens layout using the smallest architecture-compatible change: one pure text formatter plus coherent existing metric constants. It preserves truthful official quota semantics, five stable rows, the existing Tk ownership model, and measured geometry contracts.
