# v0.7.1 Activity Localization and Compact Geometry Correctness Implementation Plan

> **For agentic workers:** Use TDD for each task. Steps use checkbox syntax.

**Goal:** Release semantic activity localization and correct manual Compact geometry/persistence.

**Architecture:** Keep activity parsing, presentation, compact geometry, and Tk ownership separate. Semantic keys travel from the activity API to the snapshot; main-window mode geometry consumes canonical settings; the pure geometry module owns compact/canonical conversion.

**Tech Stack:** Python 3, Tkinter, unittest, existing Windows probes.

## Global constraints

- Supported companion languages remain `en` and `zh-CN`.
- Keep five row IDs and all quota/battery/no-fallback contracts unchanged.
- `x/y` mean canonical expanded position; Compact is manual and persisted.
- Use no dependency or broad UI refactor.
