# v0.6.3 Battery Quota Source Selector Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add one persisted binary source selector for the existing shared ten-cell battery.

**Architecture:** Configuration normalizes `battery_quota_source`; presentation selects exactly one classified quota window before the existing `battery_presentation()` function; both BatteryView modes consume that one result.

**Tech Stack:** Python, Tkinter, unittest, existing Windows release gates.

## Global Constraints

- Legal values: `primary_5h` and `weekly`; missing or invalid means `weekly`.
- No fallback, automatic source movement, third state, extra status row, dependency, or duplicated battery semantics.
- Row visibility is independent; preserve existing transactional settings, compact, DPI, Shell, and five-row contracts.
