# v1.0.2 ZIP First-Launch Installation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the extracted versioned product ZIP install and launch its exact local version through `launch.cmd`, while preserving the verified public bootstrap workflow and publishing v1.0.2.

**Architecture:** `launch.cmd` routes configured directories to `launch.ps1` and unconfigured extracted directories to a new mutually exclusive `install.ps1 -SourceRoot` mode. Both local-source and verified-artifact inputs converge on the existing staged installation transaction so Python selection, private dependencies, rollback, shortcuts, and startup remain single-sourced.

**Tech Stack:** Windows CMD, PowerShell 5.1+, Python 3.10+ standard library, `unittest`, GitHub Actions, GitHub Releases.

## Global Constraints

- Target version is exactly `1.0.2` / tag `v1.0.2`.
- Windows 11 x64 remains the supported platform; Windows 10 remains best-effort.
- Add no production dependency and do not change application UI or quota behavior.
- The public bootstrap retains exact Release ZIP plus SHA-256 verification.
- The local ZIP workflow installs the extracted version and performs no GitHub resolution or repeat ZIP download.
- Preserve unrelated untracked `docs/assets/promo/` and `scripts/create_youtube_shorts.ps1`.
