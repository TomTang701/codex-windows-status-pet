# v0.5.5 Mixed-DPI Startup Position Recovery Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Preserve a saved position that is legal at its target monitor DPI during startup, without weakening recovery of genuinely invalid positions.

**Architecture:** `Pet.safe_position()` currently derives containment dimensions from the withdrawn bootstrap HWND DPI. Add one pure monitor-selection helper to the existing display API, then derive only the startup recovery containment metrics from the monitor containing the saved point. The actual target-window DPI sync and all geometry/font authority remain unchanged after the recovery decision.

**Tech Stack:** Python 3, Tk, Win32 monitor snapshots, `unittest`.

```python
def monitor_for_point(x, y, monitors=None):
    monitors = monitor_snapshot() if monitors is None else monitors
    for monitor in monitors:
        left, top, right, bottom = monitor.get("work", ())
        if left <= x < right and top <= y < bottom:
            return monitor
    return None
```
