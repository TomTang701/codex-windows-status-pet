# v0.4.1 Correctness Stabilization Implementation Plan

> **For agentic workers:** Execute sequentially in the current session. Tom's allowed-skill policy does not authorize additional execution skills by default. Use `test-driven-development` for every production behavior change.

**Goal:** Eliminate minimum-scale reset-row clipping and route tray/quota transport failures through the authoritative five-row presentation path.

**Architecture:** Keep `derive_window_metrics` as the sole expanded metric source, preserve canonical geometry, and use measured content-safe vertical padding at 80% and 95%. Extend the existing pure status snapshot and presentation controller rather than adding a new subsystem. Tk integration tests exercise mapped production widgets at all 25 scale steps and actual queue paths; the UI applies all results with `StatusRows.configure_rows`.

**Tech Stack:** Python 3.12, `unittest`, Tkinter, existing pure API/controller/UI modules.

## Global Constraints

- Supported window scales remain exactly 80–200% in 5% steps.
- Expected expanded geometry remains 264×110, 330×138, 495×207, and 660×276 at 80/100/150/200%; vertical padding is 7 px at 80% and 8 px at 95%.
- Preserve five row IDs: `activity`, `progress`, `primary_5h`, `weekly`, `reset_credit`.
- No new module, dependency, configuration field/schema, network/provider, worker, process, polling loop, telemetry, feature, or broad refactor.
- Raw exception text must not be rendered.
- Workers enqueue only; all Tk updates remain on the main thread.
- No remote Git/GitHub action is authorized by this plan.

- [ ] **Step 1: Write the failing mapped-Tk regression test**

  Create `tests/test_ui_content_fit.py` using the existing `runpy`/`DummyServer`/`DummyTray` pattern from `tests/test_ui_menu.py`. The focused assertion body must be:

  ```python
  APPROVED_ROWS = {
      "activity": "Codex 输出中",
      "progress": "活动对话 1 个",
      "primary_5h": "5h 10% / 17:23",
      "weekly": "周 86% / 12:23 7/17",
      "reset_credit": "重置 5 次 / 18:40 7/12",
  }

  def test_supported_scales_allocate_every_requested_row(self):
      for scale in range(80, 201, 5):
          geometry = (round(330 * (scale / 100)), round(138 * (scale / 100)))
          with self.subTest(scale=scale):
              app = self.new_app(scale)
              try:
                  app.text.configure_rows(rows=APPROVED_ROWS)
                  app.update()
                  self.assertEqual((app.winfo_width(), app.winfo_height()), geometry)
                  self.assertLessEqual(app.winfo_reqheight(), app.winfo_height())
                  self.assertLessEqual(app.text.winfo_reqheight(), app.text.winfo_height())
                  for row_id, label in app.text.labels.items():
                      self.assertGreater(label.winfo_height(), 0, row_id)
                      self.assertLessEqual(label.winfo_reqheight(), label.winfo_height(), row_id)
                      self.assertLessEqual(label.winfo_y() + label.winfo_height(), app.text.winfo_height(), row_id)
                  reset = app.text.labels["reset_credit"]
                  self.assertLessEqual(reset.winfo_reqwidth(), reset.winfo_width())
                  self.assertLessEqual(reset.winfo_x() + reset.winfo_width(), app.text.winfo_width())
              finally:
                  self.destroy_app(app)
  ```

  `new_app(scale)` must instantiate the production facade, apply a temporary settings path, set scale and safe coordinates, and map with `update()`. `destroy_app` must cancel Tk callbacks, shut down the application controller, destroy the root, and collect garbage as existing UI tests do.
