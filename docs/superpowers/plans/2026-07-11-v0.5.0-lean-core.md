# v0.5.0 Lean-Core Simplification Implementation Plan

> **For agentic workers:** Implement task-by-task with the allowed `superpowers:test-driven-development` workflow; do not invoke unapproved execution skills.

**Goal:** Remove three zero-consumer modules and one pass-through quota wrapper while preserving all product behavior and release contracts.

**Architecture:** Keep the existing strict quota parser and connect the main-window worker directly to it. Delete superseded resize/model files and implementation-detail tests, add one negative source-boundary contract, and update bilingual API documentation and measured release evidence.

**Tech Stack:** Python 3.11+, `unittest`, Tk/Win32 existing integration tests, Markdown/JSON governance.

## Global Constraints

- No configuration schema, UI behavior, provider, network, dependency, polling, telemetry, or credential-boundary change.
- Keep schema-1 downgrade `font_size`, `window_width`, `window_height`, and `scale_mode` fields.
- Preserve strict quota field allowlisting, malformed handling, token/unknown-field exclusion, last-good/stale behavior, and five stable rows.
- English normative documents and Chinese translation pairs change together.
- Measure reductions from the recorded 53 production files, 3,038 production LOC, 32 API modules, and 2 dependencies.
