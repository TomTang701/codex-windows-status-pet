# v0.8.0 Productization and Menu Unification Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans task-by-task. Steps use checkbox syntax for tracking.

**Goal:** Release a Windows 11 x64 onedir application with unified tray/overlay menu semantics and a safe per-user install lifecycle.

**Architecture:** A pure menu-model API is the shared semantic authority; Tk and pystray remain surface adapters. One PyInstaller onedir spec and one builder construct a strict runtime root; PowerShell scripts validate only that artifact before touching the app-owned per-user directory.

**Tech Stack:** Python 3.11+, Tkinter, pystray, Pillow, PyInstaller, PowerShell 5.1+, Windows 11 x64, unittest.

## Global constraints

- Preserve all v0.7.1 runtime behavior except menu unification.
- Use PyInstaller onedir; no new packaging framework, updater, startup entry, telemetry, or service.
- The release package contains only runtime files, manifest, LICENSE, notices, and uninstaller; never repository tests, docs, Goal, CI, source tools, settings, credentials, or sessions.
- All runtime state transitions remain on the Tk owner; the pystray thread queues stable action IDs only.
- All evidence labeled packaged must come from the packaged executable; Sandbox/VM evidence is not physical-hardware evidence.
