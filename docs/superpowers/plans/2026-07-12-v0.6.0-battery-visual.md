# v0.6.0 Segmented Battery Visual Implementation Plan

> **For agentic workers:** Execute inline under the active Program Goal. Use TDD before each production behavior change and verify every task before committing.

**Goal:** Replace the paw with a truthful, interactive 2×5/10-cell battery in expanded and compact modes.

**Architecture:** Add one pure battery-presentation authority for remaining percentage, ceil segment count, ordered colors, and unavailable state. Add one focused Tk Frame view with ten cells that consumes it in both modes. Preserve the five-label StatusRows architecture.

**Tech Stack:** Python 3, unittest, Tk Frame/Label grid, existing DPI/content-fit probes.

## Constraints

- Exactly five status row identities; no sixth row.
- Remaining semantics: `clamp(100 - usedPercent, 0, 100)` and `ceil(remaining/10)` except zero.
- Fixed red/orange/yellow/light-green/strong-green bands; segments 9–10 use a different stronger green from 7–8.
- No paw, Canvas, emoji, animation, dependency, provider, configuration, or generic renderer.
