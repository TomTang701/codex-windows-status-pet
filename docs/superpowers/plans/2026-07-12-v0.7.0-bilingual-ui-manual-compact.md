# v0.7.0 Bilingual UI and Manual Compact Implementation Plan

> **For agentic workers:** Use the approved TDD workflow task-by-task. Steps
> use checkbox syntax for tracking.

**Goal:** Ship the approved documentation hygiene, English/Simplified Chinese
runtime UI, and persistent manual Compact behavior as v0.7.0.

**Architecture:** Keep the existing configuration, presentation, Tk adapter,
and tray ownership boundaries.  Add two small document gates and one pure
localization API; make `Pet` consume normalized state without introducing a
generic language or layout framework.

**Tech Stack:** Python 3, Tkinter, pystray, unittest, existing repository
quality/release scripts.

## Global Constraints

- Documentation bilingual scope is exactly `docs/document_manifest.json` pairs.
- Every tracked Markdown document must avoid local absolute paths above
  `codex-windows-status-pet/`.
- Supported runtime values are only `en` and `zh-CN`; default is `en`.
- `compact` is an additive persisted boolean defaulting to `false`.
- Legacy `compact_when_idle` never implies manual compact state.
- Preserve five row identities, ten-cell battery, selected battery source,
  no-fallback semantics, Settings transactions, display recovery, tray, and
  Shell identity.
