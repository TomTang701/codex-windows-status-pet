# v0.6.1 Quota Window Identity Implementation Plan

**Goal:** Classify safe official quota windows by duration so 5-hour and weekly values cannot leak into each other's rows or the fixed weekly battery.

**Architecture:** `quota_parse_api` will retain only `usedPercent`, `resetsAt`, and `windowDurationMins`, then map 300-minute windows to the existing 5-hour slot and 10080-minute windows to the existing weekly slot. `status_snapshot_api` will consume weekly only for the v0.6.1 battery.

## Constraints

- No auth/token access, third-party endpoint, raw-key identity, fallback, sixth row, or new framework.
- Preserve existing transport, QuotaState, five rows, battery geometry, mixed-DPI, and Shell identity.
