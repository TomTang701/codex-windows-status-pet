# v0.5.3 Shell Identity Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Restore the overlay's Windows tool-window shell identity without changing its visible desktop or notification-area behavior.

**Architecture:** The defect is an indirect Tk lifecycle effect: an early native mapping establishes `WS_EX_TOOLWINDOW`, then `-alpha` replaces the extended style before final mapping. Normalize the *real root HWND* immediately after every `deiconify()` boundary using a small Windows-only runtime helper. The helper adds `WS_EX_TOOLWINDOW`, removes `WS_EX_APPWINDOW`, and requests a non-activating frame refresh; it does not change ownership, taskbar APIs, layout, or settings.

**Tech Stack:** Python 3, Tk, Win32 `user32.dll` through `ctypes`, `unittest`.

## Global Constraints

- Scope is limited to the v0.5.3 Shell Identity design: no battery, layout, paw, installer, or productization work.
- Preserve desktop visibility, tray reachability, transparency, topmost behavior, and the v0.5.1 geometry guarantees.
- Verify the actual `GetAncestor(widget_hwnd, GA_ROOT)` window, never only Tk `overrideredirect()` state.
- On Windows the root HWND must have `WS_EX_TOOLWINDOW` and not have `WS_EX_APPWINDOW` after all required lifecycle transitions.
- No new dependency, owner-window technique, taskbar mutation API, or release version change before Quality/RC approval.

### Task 2: Normalize Shell identity at the mapping boundary

**Files:**
- Modify: `scripts/api/runtime_api.py`
- Modify: `scripts/ui/main_window.py`
- Test: `tests/test_ui_shell_identity.py`

**Interfaces:**
- Consumes: `ensure_overlay_toolwindow(widget_hwnd: int) -> bool` from `scripts/api/runtime_api.py`.
- Produces: a Windows-only idempotent root-HWND normalizer, called immediately after `Pet.deiconify()` in initial construction and `Pet.show_window()`.

- [ ] **Step 1: Implement the smallest Windows helper**

```python
def ensure_overlay_toolwindow(widget_hwnd: int) -> bool:
    root = user32.GetAncestor(widget_hwnd, GA_ROOT)
    style = get_window_long_ptr(root, GWL_EXSTYLE)
    target = (style | WS_EX_TOOLWINDOW) & ~WS_EX_APPWINDOW
    if target != style:
        set_window_long_ptr(root, GWL_EXSTYLE, target)
        user32.SetWindowPos(root, 0, 0, 0, 0, 0,
                            SWP_NOMOVE | SWP_NOSIZE | SWP_NOZORDER |
                            SWP_NOACTIVATE | SWP_FRAMECHANGED)
    return bool(get_window_long_ptr(root, GWL_EXSTYLE) & WS_EX_TOOLWINDOW)
```

Guard non-Windows or unavailable Win32 APIs by returning `False`; do not throw during normal overlay startup.

- [ ] **Step 2: Apply it only after native mapping boundaries**

```python
self.deiconify()
ensure_overlay_toolwindow(self.winfo_id())
```

Use the same two lines after `deiconify()` in `show_window()`. Do not add an owner, `WS_EX_APPWINDOW`, taskbar API, timer, or settings value.
