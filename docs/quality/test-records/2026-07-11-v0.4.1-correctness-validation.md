## Invalidated evidence and reproduced RED

The first mapped-Tk test instantiated `Pet` without the production `enable_dpi_awareness()` call. It therefore measured at 96 DPI and incorrectly passed after a padding-only change. Tom's production screenshot then showed the Reset Credit row clipped and invalidated that completion claim.

The corrected production-order probe enabled per-monitor DPI awareness before creating Tk. At effective DPI 120, the old implementation clipped all 25 supported scale steps. At 80%, the root was 264×110 physical pixels while Tk requested 205×129; the Reset Credit row received 4 px while requesting 23 px.

Root cause: Tk point fonts already scale for DPI, but geometry, padding, the face/text gap, and wrap length were still treated as unscaled physical pixels. The old test did not exercise this mixed-unit production path.

The error-path RED cases were independent:

- a `tray_error` action sent Label-only options to the `StatusRows` frame, raised before recovery scheduling, and left all rows unchanged;
- a current-generation quota transport failure hit the same invalid configuration path after domain state became unavailable.
