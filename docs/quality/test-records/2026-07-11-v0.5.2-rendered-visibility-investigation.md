# v0.5.2 Rendered Visibility Investigation

## Conclusion

The supplied screenshot is authoritative evidence of clipping, but machine provenance proves that its process was not running released v0.5.1. PID 7164 started from the repository at 19:05:52, before the v0.5.1 production fix was committed at 19:20 and merged at 19:27. The process retained pre-fix code in memory while the files beneath its command line were later updated by Git.

After terminating only that project process and launching the same root launcher from the verified v0.5.1 source, PID 21944 started at 20:20:57. Its startup log recorded target `window_dpi=96`; its HWND client was `264x110` at `(4216,1276)-(4480,1386)`. A host capture of the exact client showed all five real production rows, including the complete `reset_credit` row.

Therefore the released-v0.5.1 failure required by the proposed Goal has not been reproduced. No production fix is permitted.

## Exact false-positive mechanism

The incident is a process-provenance false attribution:

```text
old process starts before fix
→ Git updates files at the same command-line path
→ inspection reads current APP_VERSION/current HEAD from disk
→ old in-memory Python process is mislabeled as v0.5.1
→ its clipping screenshot is interpreted as a v0.5.1 counterexample
```

The 15 lifecycle checks passed because they exercised a newly constructed candidate `Pet`, not PID 7164. RC remained green because it validates a checkout/build candidate and does not mutate or certify an already-running user process. The old geometry checks also have a rendered-glyph blind spot, but that blind spot cannot be named as the cause of this screenshot without a v0.5.1 reproduction.

## Minimum authority approaches

### A. Corrected Tk/native rendered-line measurement

Potentially useful because requested/allocated geometry is not rasterized-glyph proof. It cannot currently supply the mandatory RED because verified v0.5.1 renders the fifth row completely in the reproduced state.

### B. Content-derived expanded height

Not justified. There is no evidence that released v0.5.1's fixed height fails after correct process restart, so changing height would be another speculative geometry patch.

### C. Minimal HWND/client capture

The least complex incident authority. It directly distinguished the stale pre-fix process screenshot from the restarted v0.5.1 client and verified the full fifth-row render. It currently passes v0.5.1, so it cannot be converted honestly into a released-v0.5.1 RED.
