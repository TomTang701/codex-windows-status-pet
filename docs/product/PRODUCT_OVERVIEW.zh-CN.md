# Codex Windows 状态宠物——产品评审

English: [English version](PRODUCT_OVERVIEW.md)

## 当前定位

这是一个面向重度 Codex 用户、低干扰的外部状态伴侣。它提供活动对话状态、额度窗口、重置次数、设置和托盘恢复路径，不修改 Codex 核心文件，也不向内置宠物注入文字。

## 当前产品契约

- 悬浮窗只显示活动对话数量，不显示计划步骤 `N/M` 文本。
- 设置严格提供两个滑块：透明度和等比例窗口大小。80–200% 的尺寸来源会一起缩放固定比例几何、字体、换行和间距；位置、颜色、置顶、锁定、Compact 和 1–10 秒刷新间隔保持不变。
- 悬浮窗隐藏或移出屏幕时，托盘是恢复路径。
- `v0.9.2-beta.1` 的 Signal HUD 展开状态最多显示四行：活动进度、5 小时额度、每周额度和重置额度；底层仍保留五个稳定行身份，活动身份由标题/状态区域承载。收缩状态显示带百分比的十格电池。
- 额度文字统一使用设置中的字体颜色；额度健康状态只通过进度条和电池颜色表示。
- beta 候选已增加设置生命周期后的 Windows 工具窗口身份规范化，但在进入 1.0 前仍需要新鲜的实体 Save/Close/右上角关闭和任务栏观察证据。
- API 边界和无界面回归测试定义在 `docs/architecture/API_SPEC.zh-CN.md`。
- 菜单定位和额度日期格式化是可独立测试的 API 边界。

## 优势

- 本地数据边界，不使用第三方后端。
- 额度数据源边界仅限本地，并拒绝传播凭据字段。
- 外部架构降低对 Codex 核心行为的影响风险。
- 明确的 schema 1 配置迁移/降级兼容性和多显示器坐标支持。
- API 拆分使配置、活动状态、菜单几何和额度格式化无需启动 Tk 即可测试。

## 当前风险

- Windows 11 x64 的 Tk 和托盘行为仍需要混合 DPI、任务栏边缘和干净机器证据；Windows 10 已延后且不声明支持。
- 托盘故障注入和单次恢复策略已经有确定性测试覆盖；直接托盘交互仍受环境限制。
- 已针对代表性五行 Tk 内容测量 80–200% 支持范围；未来更长的本地化文本仍需有界文字评审。
- 干净虚拟环境和 Windows CI runner 已通过依赖安装及回归测试；实体混合 DPI 覆盖仍未完成。
- quota-float 的视觉模型值得借鉴，但没有复制其基于令牌的外部额度数据边界。

## 建议的产品优先级

1. 完成 v0.9.2-beta.1 的打包 UI、任务栏和双语证据。
2. 在硬件可用时继续补充真实的混合 DPI 和独立干净机器证据。
3. 在不扩展菜单或隐私边界的前提下增加无障碍/键盘覆盖。
4. 保持默认悬浮窗紧凑、聚焦状态；除非明确做成可配置功能，不要把计划步骤详情重新放回主信息行。
